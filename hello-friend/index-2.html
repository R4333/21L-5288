About

2021-11-26 — Written by R433 — 3 min read

Hi there

My name is R433, or Muhammad Abdullah and I’m the author of this blog. I made it to help me present my ideas better.

We all know how hard is to start something on the web, especially these days. You need to prepare a bunch of stuff, configure them and when that’s done — create the content.

This site is pretty basic and covers all of the essentials. All I have to do is start typing!

This site was a failed attempt at creating a blog a few month ago, which I had to abandon as I wasn't able to afford hosting on a MEAN Stack based host. In the end I resorted to hosting a wordpress based site on a free LAMP stack shared hosting service and this blog was left collecting dust.

The basic layout of the blog was created with SASS and javascript using node.js, with code collected from several sources, which was then mish-mashed into this blog. The sites stylesheet was then integrated with HUGO which is a open-source headless Content Management System and a static site generator, using which the posts in the blog were created. It was installed as a npm package in node.js. It was done so to ease the process of making different pages for the blog posts and to save time. These HTML pages were then transplanted to my sites directory structure to integrate it with the landing page/portfolio.

The landing page/Portfolio was created using pure CSS and HTML. Bootstrap CSS and JS libraries were used. Bootstrap was implemented using the CDN method. The whole site was created in a span of hours, overnight. For the portfolio page, help was taken from bootstraps official documentations and examples. otherthan that, I tried to use Randomuser.me API to get random information for the team section of the page but I couldn't properly implement it. In the end, I just href-ed user images from the Randomuser.me's database and put placeholder text. I also implemented a map using mapbox which is also not properly implemented as it's not centering the map on the desired location but onto boston where mapbox's parent company is based in. I changed the centering coordinates to someplace in antarctica but it still isn't working. I think these scripts need a working node.js back-end to work. it is out of the scope of this assignment. My mapbox API key/Access Token is embedded into the page, I hope you don't extract the Access Token, send requests with that key and lapse my requests quota on mapbox. lol.

I plan to implement a JS based in-browser PlayStation 1 or GBA or some other retro console emulator into the site. right now, the candidate in my eyes is eNGE PS1 emulator.

This site is an ode to Sam Esmail's TV series, Mr. Robot.

The site includes:

  • light/dark mode, depending on your preferences
  • great reading experience thanks to Inter font, made by Rasmus Andersson
  • nice code highlighting thanks to PrismJS
  • integrated a JS implementation of Conway's Game of Life in the site.
  • integrated JS based Skifree Game

So, there you have it… enjoy!
